  • Triple View: This is task #1, the following are views loaded dynamically through state parameters:
    • /triple-view/admin: This content is loaded when the param is 'admin', it also has an optional second parameter that populates an additional view. In this simple example it just reflects whatever the param is, for example visiting /triple-view/admin/additional will show "additional"
    • /triple-view/secret: This content is loaded when the param is 'secret'
    • /triple-view/anything: This content is loaded when the param is anything other than the other two
  • Communication between controllers: This is task #2, the view for Input One has a link to the view for Input Two. By filling out the input in Input One, a service will keep track of the data. When Input Two gets loaded it uses the same service to get the data set in Input One.
  • Task #3 is having page transition animations, which can be observed when navigating to any of the links
  • Sortable list: This is task #4, an array of 50 items with random values gets generated and it is sortable by fields when clicking on the column headers.

  • Sum of three inputs: This is task #7. The sum of inputs 1 through 3 is displayed in input 4. When input 4 gets updated the difference is distributed in a weighted manner across the other 3 inputs.
